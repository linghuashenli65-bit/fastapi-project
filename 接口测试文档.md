## 概述

本文档定义了学生管理系统的所有API接口测试用例，涵盖学生、教师、班级、就业、成绩及AI智能分析功能。

**API基础地址**: `http://localhost:8000`

**认证方式**: JWT Token（登录接口除外，所有业务接口需在请求头携带 `Authorization: Bearer <token>`）

**数据格式**: JSON

**统一响应格式**: 所有业务接口均采用 `UnifiedResponse` 格式

**成功响应:**
```json
{
  "status": 1,
  "messages": "操作成功",
  "datas": [...],
  "pagination": { "count": 100, "page": 1, "page_size": 10, "total_pages": 10 }
}
```

**失败响应:**
```json
{
  "status": 0,
  "messages": "错误信息",
  "datas": null,
  "pagination": null
}
```

> 认证接口（`/auth/jwt/login` 等）返回原始格式，不走统一响应。

## 学生模块

### 1. 分页获取学生列表

| 项目 | 内容 |
|------|------|
| 接口地址 | `GET /student/` |
| 功能描述 | 分页获取学生列表，支持按姓名模糊搜索 |

**请求参数:**

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| page | int | 否 | 1 | 页码 |
| size | int | 否 | 10 | 每页数量 |
| name | string | 否 | - | 学生姓名（模糊搜索） |

**测试用例:**

#### 测试用例 1.1 - 获取第一页学生列表

```http
GET /student/?page=1&size=10
```

**预期结果:**
```json
{
  "status": 1,
  "messages": "查询成功",
  "datas": [
    {
      "id": 1,
      "student_no": "230900001",
      "name": "张三",
      "gender": "男",
      "birth_date": "2000-01-01",
      "birthplace": "北京",
      "graduated_school": "北京市第一中学",
      "major": "计算机科学与技术",
      "enrollment_date": "2024-09-01",
      "graduation_date": null,
      "education": "本科",
      "consultant_no": null
    }
  ],
  "pagination": {
    "count": 100,
    "page": 1,
    "page_size": 10,
    "total_pages": 10
  }
}
```

#### 测试用例 1.2 - 按姓名搜索学生


**预期结果:** 返回姓名中包含"张"的学生列表

### 4. 创建学生

| 项目 | 内容 |
|------|------|
| 接口地址 | `POST /student/` |
| 功能描述 | 创建新学生信息 |

**请求体:**

```json
{
  "name": "李四",
  "gender": "M",
  "birth_date": "2001-05-15",
  "birthplace": "上海",
  "graduated_school": "上海市第一中学",
  "major": "软件工程",
  "enrollment_date": "2024-09-01",
  "education": 3,
  "consultant_no": "T001"
}
```

**字段说明:**

| 字段名 | 类型 | 必填 | 说明 | 可选值 |
|--------|------|------|------|--------|
| name | string | 是 | 学生姓名 | - |
| gender | string | 否 | 性别 | M/F |
| birth_date | date | 否 | 出生日期 | ISO 8601格式 |
| birthplace | string | 否 | 出生地 | - |
| graduated_school | string | 否 | 毕业院校 | - |
| major | string | 否 | 专业 | - |
| enrollment_date | date | 是 | 入学日期 | ISO 8601格式 |
| graduation_date | date | 否 | 毕业日期 | ISO 8601格式 |
| education | int | 否 | 学历代码 | 1-5 |
| consultant_no | string | 否 | 咨询师编号 | - |

**学历代码对照表:**

| 代码 | 学历 |
|------|------|
| 1 | 高中 |
| 2 | 大专 |
| 3 | 本科 |
| 4 | 硕士 |
| 5 | 博士 |

**测试用例:**

#### 测试用例 4.1 - 创建学生（必填字段）

```http

{
  "name": "王五",
  "enrollment_date": "2024-09-01"
}
```

**预期结果:** 创建成功，返回学生信息，HTTP状态码 200

```json
{

  "messages": "创建成功",
  "datas": [
    {
      "id": 2,
      "student_no": "230900002",

}
```

```http
  "name": "赵六",

  "major": "人工智能",
```

## 附录

### A. HTTP状态码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 404 | 资源不存在 |
| 422 | 数据验证失败 |
| 500 | 服务器内部错误 |

### B. 统一响应格式

**成功响应：**
```json
{
  "status": 1,
  "messages": "操作成功",
  "datas": [...],
  "pagination": null
}
```

**失败响应：**
```json
{
  "status": 0,
  "messages": "错误描述信息",
  "datas": null,
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| status | int | 状态码：1=成功，0=失败 |
| messages | string | 提示信息 |
| datas | array\|null | 数据列表 |
| pagination | object\|null | 分页信息 |

### C. 分页响应格式

```json
{
  "status": 1,
  "messages": "查询成功",
  "datas": [],
  "pagination": {
    "count": 100,
    "page": 1,
    "page_size": 10,
    "total_pages": 10
  }
}
```

### D. 日期格式说明

所有日期字段均使用 **ISO 8601** 格式：`YYYY-MM-DD`

示例：
- `2024-09-01` - 2024年9月1日
- `2000-01-01` - 2000年1月1日

### E. 测试建议

1. **单元测试**: 对每个接口的单一功能进行测试
2. **集成测试**: 测试模块间数据关联和业务流程
3. **边界测试**: 测试参数边界值、特殊字符等
4. **性能测试**: 对分页、统计等接口进行压力测试
5. **安全测试**: 测试SQL注入、XSS等安全问题

### F. 常见问题

**Q: 如何测试分页功能？**

A: 依次请求 page=1, page=2, ...，直到返回的items为空，验证total和pages是否正确。

**Q: 如何测试软删除？**

A: 删除后再次查询该资源，应该返回404，但数据库中deleted_at字段被更新。

**Q: AI接口响应慢怎么办？**

A: AI接口依赖外部LLM服务，响应时间不确定。建议使用异步方式处理或设置超时。
